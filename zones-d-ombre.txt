### **Interface Employé : Liste de Tâches (ex. "Peser les porcs")**

#### **Comment cela affecte le programme ?**
Dans l'interface employé, la liste de tâches comme "peser les porcs" (visible dans le wireframe #4 "Gestion tâches" pour le côté employé et suggérée dans d'autres comme #8 "Gestion Enclos et Mise en Vente") a plusieurs impacts sur le programme :
- **Mise à jour des statuts** : Lorsqu’un employé marque une tâche comme "peser les porcs" comme terminée (par exemple, en cochant une case et en cliquant sur "DONE" dans #4), le statut de la tâche passe de "en cours" à "accomplie" dans la base de données.
- **Enregistrement des résultats** : Cette tâche implique la collecte de données (par exemple, le poids des porcs). Ces données doivent être enregistrées et liées aux entités concernées (porcs, enclos, cycles reproductifs).
- **Effets en cascade** : Les données de poids peuvent influencer d'autres parties du système, comme :
  - **Santé** (#10 "Gestion Santé") : Le poids peut signaler des problèmes de santé (ex. sous-poids nécessitant un diagnostic).
  - **Reproduction** (#11 "Gestion Reproduction") : Le poids peut affecter la décision d’insémination ou la gestion des portées.
  - **Vente** (#8 "Gestion Enclos et Mise en Vente") : Les porcs atteignant un poids cible peuvent être marqués "Prêt à être vendu".
- **Mise à jour des stocks** : Si la tâche implique l’utilisation de matériel (ex. balance), cela pourrait affecter une table de gestion d’équipement ou d’inventaire (non visible dans les wireframes actuels).

#### **Tables nécessaires**
Pour supporter cette fonctionnalité dans la base actuelle, les tables suivantes sont nécessaires :
- **Table `bao_tache` (Tâches)** :
  - Champs : `task_id` (clé primaire), `task_name` (ex. "Peser les porcs"), `description`, `created_date`, `due_date`, `status` (ex. "pending", "completed").
- **Table `bao_tache_employe` (Assignation des tâches aux employés)** :
  - Champs : `assignment_id` (clé primaire), `task_id` (clé étrangère), `employee_id` (clé étrangère), `assigned_date`, `status`, `completed_date`, `result` (ex. poids des porcs en kg).
- **Table `bao_pesee` (Pesées)** :
  - Champs : `weighing_id` (clé primaire), `task_id` (clé étrangère), `pig_id` ou `enclosure_id` (selon la spécificité), `weight`, `date`, `employee_id`.
- **Table `bao_truie` (Truies)** ou **Table `bao_portee` (Portées)** : Pour lier les poids aux truies ou aux porcelets si nécessaire.
- **Table `bao_enclos` (Enclos)** : Pour associer les tâches à des enclos spécifiques (ex. "Poids estimé" dans #8).

#### **Zones d'ombre dans l'interface employé**
- **Assignation des tâches** : Dans #4, les tâches apparaissent dans une liste, mais il n’est pas clair comment elles sont assignées (manuellement par l’admin ou automatiquement ?).
- **Détails des tâches** : Aucun wireframe ne montre les instructions spécifiques (ex. quels porcs peser, quel équipement utiliser).
- **Suivi des progrès** : Pas de mécanisme pour indiquer une tâche "en cours" ou partiellement terminée (ex. 10 porcs pesés sur 20).
- **Notifications** : Les employés ne semblent pas recevoir d’alertes pour les tâches nouvelles ou en retard.

#### **Suggestions pour l'interface employé**
- **Mécanisme d’assignation** : Ajouter un système clair pour assigner les tâches (ex. un champ "Assigné à" dans la liste des tâches).
- **Détails des tâches** : Inclure une section "Détails" dans la liste (ex. "Peser les porcs de l’enclos E1, utiliser la balance X").
- **Suivi des statuts** : Ajouter des statuts intermédiaires (ex. "en cours") avec des indicateurs visuels (ex. couleurs).
- **Formulaire de résultats** : Après avoir marqué une tâche comme terminée, afficher un formulaire pour entrer les poids, avec validation (ex. poids > 0).
- **Notifications** : Implémenter un système d’alertes (ex. pop-up ou icône dans l’interface pour les nouvelles tâches).
